Debian Twitter

The Debian project on Twitter

Tweeting

It's possible for all Debian Developers with GPG key on the official Debian keyring to tweet under the @debianproject account.

$ echo "Hello Twitterland" | gpg --clearsign | lwp-request -m POST http://twitter.debian.net/post

As such, a tweet will appear on @debianproject's account with an appended credit/uid, such as:

Hello Twitterland (from damog)

The corresponding username is, of course, the developer's Debian username (as found on the db.debian.org LDAP).

The Identi.ca @debianproject account also gets updated simultaneously.

Neat, I want to test it!

Please, do not "try" or "test" it. Given that the @debianproject account already has a number of followers, it's better if we apply common sense and only use this service when you, as Debian developer, has something worthwhile to communicate on Twitter.

Is there a policy on what (and what not) to post?

While there is no hard and fast rule on what can be posted, restricting your posts to technical and social aspects of the Debian project is a must. A (non-exhaustive) list of some useful information which can be disseminated through this medium would be:

  • Notifications on uploads of new packages/versions with interesting or long awaited updates
  • Links to interesting articles and news tidbits about Debian in magazines
  • Links to significant posts on mailing lists such as debian-news, debian-infrastructure-announce, debian-devel-announce etc.
  • Links to interesting blog posts on Debian
  • Security advisories on Debian packages

Is @debianproject the official Debian's Twitter account?

Nothing's really official on Debian until it involves a debian.org domain, so no.

Why did you build this?

Because there are some DDs who are interested on Twitter and we'd like the Debian project to have some participation in it. Nothing more.

My GPG key is not on the Debian keyring, can I post?

Tell us why, of course it's possible. We can talk about it. Debian Maintainers in the DM keyring are now allowed to post by default.

Can I haz code?

Sure thing. The code is nothing more than a mod_perl 2 response handler with custom directives. Go grab it at GitHub, here. As it is right now, it can be easily customized to use any Twitter account with any GPG keyring.
